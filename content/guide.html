POC Implementation Guide

This is a placeholder. Replace content/guide.html with your real customer-facing implementation guide.

Hint: the file is read server-side by lib/guide.ts, which extracts the <style> blocks and the <body> contents and renders them inside the authenticated SPA shell. Inline <script> tags are stripped — Mermaid is loaded separately via next/script in app/layout.tsx, so any <div class="mermaid"> blocks will render automatically.

How to swap in your guide

  1. Drop your standalone HTML file at content/guide.html (overwriting this placeholder).
  2. If your guide uses Mermaid diagrams, ensure each diagram is wrapped in <div class="mermaid">...</div> and you can remove your own Mermaid <script> tags — the layout loads them.
  3. Set the NEXT_PUBLIC_BRAND_* env vars in Vercel to customize titles, login page text, and the browser <title>.
  4. Deploy. The portal session-gates the guide via next-auth against the Okta OIDC app provisioned by modules/poc-portal.
